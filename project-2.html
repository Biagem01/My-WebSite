INTRODUCTION

This project is a system developed in SQL to manage event ticket sales. It focuses on the conceptual, logical and physical design of a database to manage information related to the events, venues, tickets, customers, artists and promoters involved in the ticket sales process.

MAIN FEATURES

Event Management: The system allows administrators to add, edit and delete events, specifying details such as date, time, location and type of event. Ticket Management: Users can view available events and purchase tickets for desired events. The system keeps track of available seats for each event and automatically manages ticket sales. Sales Transactions: The system records ticket sales transactions, including details such as transaction ID, customer ID, event ID, quantity of tickets sold, and transaction total.

GENERAL SETTINGS

Splash Screen: A personalized splash screen that introduces the game to the user, creating an impression of anticipation and anticipation. Game Icon: A representative icon for the game, which helps identify and differentiate it within the device or gaming platform.

CONCEPTUAL DESIGN

The conceptual design aims to collect the requirements requested by the customer and define the conceptual model of the database. This includes:

  • Entities involved: : Evento, Luogo, Biglietto, Cliente, Artista, Promotore.
  • Relations: Among the entities there are relationships such as the association of an event with a location, the participation of an artist in an event, and the purchase of tickets by customers

SYSTEM REQUIREMENTS

The database system must be able to handle different types of data and information, including:

  • Event Information: Event name, date, time, description
  • Venue Information: Venue name, address, city, state, maximum capacity.
  • Ticket Information: Ticket type, price, description, quantity available.
  • Customer information: Name, surname, address, gender.
  • Information on the Artists: Name, musical genre, biography.
  • Information on the Promoters: Name, surname, telephone number.

LOGICAL DESIGN

Logical design is about translating the conceptual model into a relational database schema. The entities and relationships defined in the conceptual design are translated into tables and relational integrity constraints.

CONCLUSION

The event ticketing system is an important resource for efficiently managing the ticketing process and event attendance. Thanks to the conceptual, logical and physical design of the database, it was possible to define a complete and well-organized data structure to manage crucial information relating to events, venues, tickets, customers, artists and promoters.
